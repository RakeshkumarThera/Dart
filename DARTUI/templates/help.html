Help

Parameters Description
BuildName Also called as a Milestone.This is used to upload the results into TestLink
Install test This file contains the install tests to be executed on each cluster before individual tests are scheduled for execution
Health Check This file contains the test to check the health of the cluster before any test which require a CheckHealth
Testsets (mandatory) This file contains a list of all the testsets and tests which are planned for execution
Clusters (mandatory) A string listing out all the names of the clusters
runLabel This is used to display the results in the Dart Dash Board by JobName
releaseName RELEASE NAME ( The Release Name Example: GGR1 )
Project Name This is used to display the results in the Dart Dash Board by ProjectName. The ProjectName is also used to upload the results into TestLink
  • If the ProjectName is not specified, the scheduler assumes the run as a Private Project.
  • The results are not displayed in the Dart Dash Board unless you click the Private checkbox in the Dash Board.
branchName This field is used to refer the GIT branch used
buildNumber This is the Jenkins Build Number. If the buildUrl is given, then the BuildNumber, RevisionNumber is automatically calculated.
revisionNumber RevisionNumber is automatically calculated
dashboard This option will upload the results into the DartDashBoard. The LOAD can have values 'Public or Private' The default is not to load the results.
Example: -l 'Public'
retry This will retry a failed or aborted test again
